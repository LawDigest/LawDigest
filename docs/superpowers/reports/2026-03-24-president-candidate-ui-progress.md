# 대통령 후보 UI 진행 상황 보고

**기준 브랜치:** `feat(codex)/president-candidate-ui-task-6`  
**비교 기준:** `docs/codex/plan-election-tab`  
**작성일:** 2026-03-24

이 문서는 `선거` 탭 전체 계획 대비 `president-candidate-ui-task-6` 브랜치의 진행 상황을 정리한 보고서다.  
결론부터 말하면, 프론트엔드 화면과 상호작용은 거의 완성 단계지만, 백엔드 계약과 실데이터 연동은 아직 시작 전이다.

## 1. 완료 / 부분완료 / 미완료

### 완료

- 선거 선택 셸이 구현되어 있다.
- 지역 확인 흐름이 구현되어 있다.
- 수동 지역 선택 UI가 구현되어 있다.
- 대통령 선거 전용 후보자 중심 화면이 구현되어 있다.
- 지역 결과 화면이 브레드크럼과 depth 탐색을 포함해 동작한다.
- test 환경에서는 mock fallback으로 화면이 유지된다.
- 테스트 코드와 Vitest 기반 검증이 추가돼 있다.

### 부분완료

- 지도 탐색은 실제 지도와 육각형 카토그램 전환 UI까지 들어가 있지만, 계획서에서 분리하던 `MapBreadcrumb`, `MapViewToggle`, `ElectionMapSection` 구조는 아직 독립 컴포넌트로 쪼개지지 않았다.
- 대통령 선거의 후보자 보기와 지역 결과 보기는 구현돼 있지만, 공약 상세를 최상위 탭이 아니라 후보 상세 안에 넣는 설계는 아직 mock 중심이다.
- 선거 목록 정렬과 비교 규칙은 반영됐지만, 백엔드가 주는 계약을 그대로 소비하는 구조는 아직 아니다.

### 미완료

- 백엔드 `ElectionController`, `ElectionService`, `ElectionFacade`, `ElectionRegionResolverService`가 없다.
- `/v1/election/*` 실제 API 계약이 없다.
- SGIS/NEC 기반 지역 추정과 결과 조회가 연결되지 않았다.
- plan 문서에서 예정한 전체 QA와 반응형 마감 정리는 아직 끝나지 않았다.

## 3. 프론트엔드 기준 진행률

### 전체 기준

- 전체 계획 대비 진행률: 약 70%
- 프론트엔드만 기준 진행률: 약 85% 이상
- 백엔드 포함 엔드투엔드 기준 진행률: 약 50% 이하

### 근거

- `ElectionShell.tsx`가 단순 라우팅 셸이 아니라 지역 확인 상태 머신까지 포함하고 있다.
- `PresidentialElectionView.tsx`와 `RegionalElectionView.tsx`가 각각의 핵심 화면을 담당하고 있다.
- `apis.ts`와 `mock.ts`가 test 환경 fallback까지 제공해 UI 검증이 가능하다.
- `compareRules.ts`, `electionLabels.ts`, 테스트 파일들이 들어가 있어 UI 로직의 대부분이 정리됐다.
- 다만 backend 구현이 비어 있어서, 실서비스 데이터와 연결된 완결성은 아직 없다.

### 해석

이 브랜치는 "화면을 먼저 완성하고, 이후 백엔드를 붙이는" 순서에 맞는 상태다.  
지금 시점에서 main에 머지한 뒤 백엔드 구현을 시작하는 전략이 적절하다.
