# 코리아리서치인터내셔널 여론조사 파서 설계

작성일: 2026-04-11

## 1. 목표

`(주)코리아리서치인터내셔널` PDF 58건을 `unparseable` 상태에서 제거하고, 기관 전용 메인 파서 1개로 전부 처리한다.

이번 작업은 `find_tables()` 기반 표 인식이 아닌, ASCII-art 형태의 텍스트 표를 복원하는 새 파싱 경로를 도입하는 것이 핵심이다.

## 2. 확인된 사실

- 전국 기준 `(주)코리아리서치인터내셔널`은 58건이며 전부 `무선전화면접` 포맷이다.
- 서울 샘플 2건 기준으로 PDF 텍스트 추출은 가능하며 `문N`, `<표 N>`, `전 체`, `BASE:전체`, `----------------------------` 구분선이 그대로 남는다.
- `page.find_tables()`는 표를 0개로 반환하므로 기존 `BaseTableParser` 흐름만으로는 파싱할 수 없다.
- 대표 스폰서 분포는 전주MBC 계열 37건, 광주MBC 계열 11건, 서울 MBC 3건, 제주MBC 계열 2건 등으로 지역별 편차는 있으나 조사기관 포맷 자체는 동일 계열일 가능성이 높다.

## 3. 범위

포함 범위:

- 코리아리서치인터내셔널 전용 파서 클래스 추가
- ASCII-art 텍스트 표 복원 유틸 추가
- 기관 등록 및 `unparseable` 제거
- 대표 PDF 픽스처 및 단위/통합 테스트 추가
- 전체 probe/coverage 기준 58건 정상 처리 확인

제외 범위:

- 다른 조사기관 포맷 일반화
- OCR 또는 외부 모델 도입
- UI/배포 작업

## 4. 설계 원칙

- 파서의 주 입력은 `full_text`이며, `fitz words`는 열 경계 보정 fallback으로만 사용한다.
- `parser.py`에는 기관별 규칙만 두고, ASCII-art 표 복원 로직은 별도 유틸 모듈로 분리한다.
- 표 전체를 완전한 일반 테이블로 복원하려 하지 말고, 이번 시스템에 필요한 `질문 제목`, `전체 행`, `선택지`, `비율`, `사례수`만 안정적으로 읽는다.
- 실패는 조용히 숨기지 않고 로그와 테스트에서 즉시 드러나게 한다.

## 5. 목표 구조

### 5.1 파서 계층

- `services/data/src/lawdigest_data/polls/parser.py`
  - `_KoreaResearchInternationalParser` 추가
  - `NEEDS_FITZ_WORDS = True` 사용
  - `parse(pages_data)`를 직접 구현
- `services/data/config/parser_registry.json`
  - 코리아리서치인터내셔널 등록 추가
  - `unparseable` 항목 제거

### 5.2 텍스트 표 유틸 계층

- `services/data/src/lawdigest_data/polls/text_table_utils.py`
  - 텍스트 정규화
  - 구분선 기반 표 블록 추출
  - 멀티라인 헤더 병합
  - 라인 기반 열 경계 추정
  - `fitz words` 기반 경계 보정
  - 전체 행 파싱 보조 함수

이 분리 구조를 통해 기관별 규칙과 텍스트 표 복원 알고리즘을 독립적으로 테스트할 수 있게 한다.

## 6. 데이터 흐름

1. 페이지 `full_text`를 읽어 제어문자와 비정상 공백을 정규화한다.
2. `<표 N>` 및 `문N` 또는 `문N-M` 패턴으로 질문 블록을 찾는다.
3. 이어지는 `-----` 구분선 쌍을 이용해 ASCII-art 표 블록을 추출한다.
4. 첫 번째와 두 번째 구분선 사이 헤더를 병합해 선택지 컬럼 라벨을 만든다.
5. 공백 폭과 숫자 정렬로 열 경계를 추정한다.
6. 경계가 불안정하면 같은 페이지의 `fitz words` x좌표로 해당 열만 보정한다.
7. `전 체` 행에서 완료 사례수, 비율, 가중값 적용 사례수를 읽는다.
8. 선택지/비율 길이를 맞춘 뒤 기존 `QuestionResult`로 변환한다.
9. 최종 결과는 기존 시스템 일관성을 위해 `question_number`를 1부터 다시 부여한다.

## 7. 예외 처리

- 1차 라인 파싱 실패 시 `fitz words` 보정으로 재시도한다.
- 보정 후에도 `전체 행` 탐지 실패 또는 `선택지 수 != 비율 수`이면 질문 번호, 페이지, 헤더 원문, 전체 행 원문을 디버그 로그로 남긴다.
- validation 규칙을 통과하지 못하는 결과는 성공 처리하지 않고 테스트에서 실패하게 둔다.

## 8. 테스트 전략

### 8.1 단위 테스트

- `text_table_utils.py`에 대한 synthetic 텍스트 조각 테스트
- 멀티라인 헤더 병합, 열 경계 추정, 전체 행 파싱, words 보정 각각 검증

### 8.2 통합 테스트

- 서울 샘플 2건 포함
- 전북 MBC 계열, 광주MBC 계열, 제주MBC 계열, 전국 공통조사 대표 PDF 추가
- 픽스처 기반 속성 검증 유지

### 8.3 전체 검증

- `probe_parsers.py` 기준 코리아리서치인터내셔널 58건 전부 성공
- `analyze_parser_coverage.py` 기준 `unparseable` 제거 반영

## 9. 리스크

### 9.1 헤더 줄바꿈 변형

지역·후보명이 길어지면 헤더 줄 수와 정렬이 달라질 수 있다.

대응:

- 헤더 병합을 고정 row index에 의존하지 않고 구분선 사이 전체 영역 기준으로 수행한다.
- 선택지 수와 비율 수가 다르면 words 보정으로 재시도한다.

### 9.2 지역별 미세 포맷 차이

전북·광주·제주 계열은 스폰서 조합과 문항 구성이 달라질 수 있다.

대응:

- 대표 PDF를 각 계열에서 최소 1건 이상 확보해 픽스처를 만든다.
- 전용 파서 내부 분기는 최소화하되, 필요하면 메서드 수준 분기로 처리한다.

### 9.3 거대 파일 `parser.py`

현재 `parser.py`는 이미 커서 기관별 로직을 계속 누적하면 유지보수가 악화된다.

대응:

- 공통 ASCII-art 유틸은 새 모듈로 분리한다.
- `parser.py`에는 기관별 파서와 최소 연결 코드만 둔다.

## 10. 완료 기준

- 코리아리서치인터내셔널 58건이 전용 메인 파서 1개로 파싱된다.
- `parser_registry.json`의 `unparseable`에서 해당 기관이 제거된다.
- 대표 지역 PDF 픽스처 테스트와 validation 테스트가 통과한다.
- 전체 probe/coverage에서 해당 기관이 성공으로 집계된다.
