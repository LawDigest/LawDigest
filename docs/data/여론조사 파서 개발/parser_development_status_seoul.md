## 기술 노트

### 파싱 불가 판정 기준

`unparseable`로 등록하는 조건:
- fitz `find_tables()`로 테이블을 인식하지 못하는 구조 (ASCII-art, 이미지 기반 등)
- 텍스트 추출 후 컬럼 슬라이싱이 구조적으로 불가능한 경우
- 개발 시도 후 신뢰할 수 있는 파싱 결과를 얻을 수 없다고 판단된 경우

### 코드 반영 위치

- `config/parser_registry.json` → `unparseable.pollsters[]` 배열
- `scripts/polls/probe_parsers.py` → `_unparseable_names` 세트로 로드, `X` 플래그 처리

### 알려진 특이사항

- **여론조사꽃**: NotoSansKR Identity-H 인코딩으로 GID 디코딩 필요. rawdict 모드 + fontTools 사용. `_gid_cache`로 폰트 로딩 1회로 최적화.
- **CBS-KSOI**: 질문 제목이 크로스탭 페이지 위 여백(`outside_text`)에 `【 표 N 】` 형태로만 존재. 다른 파서와 달리 전체 페이지 텍스트에서 추출 불가.
- **여론조사공정(주)**: 헤더 테이블(col3~)과 데이터 테이블(col4~)의 컬럼 오프셋이 다름. `HEADER_OPT_COL=3`, `META_COLS=4`로 분리 처리.
